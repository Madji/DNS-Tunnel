DNS Tunnel

Simple scripts for running an automated DNS Tunnel system.

View project onGitHub

Introduction

DNS Tunnel is a simple set of scripts to automate the setup and maintenance of a DNS Tunnel system. Use cases include proxying country restricted services such as Netflix or bypassing censorship.

Requirements

  • A clean Debian 7 install.
  • Full iptables support (some VPS providers do not offer this!)
  • Currently the script is best used alone as it uses port 80 (HTTP), 443 (HTTPS/SSL) and 53 (DNS).
  • For Netflix or Hulu forwarding, the VPS must have a US based IP.

Setup

  1. Create a GitHub account if you do not have one already.
  2. Go to Gist and create two files. Name them host and user. It's recommended that you set the Gist to private. An example Gist can be found here.
  3. The hosts file should contain a list of hosts to forward and the user file should contain a list of IPs or hostnames to allow through the firewall (Dyanmic DNS works very well and it's recommended if your IP changes). Ensure there is a newline at the end of each file.
  4. Obtain the raw URL to your Gist and note it down. This can be found by clicking the "View Raw" icon on file in your Gist and copying the URL with everything after /raw/ removed. For example: > https://gist.githubusercontent.com/maxexcloo/10565742/raw/7be4e1bd240341786011545363cf1604bc929495/host becomes > https://gist.githubusercontent.com/maxexcloo/10565742/raw/
  5. Log into your server as the root user and run the following commands, substituting in the raw Gist address you noted previously: wget https://raw.githubusercontent.com/maxexcloo/DNS-Tunnel/master/install.sh bash install.sh NOTEDGISTURLHERE dnstun NOTEDGISTURLHERE
  6. Restart your server to confirm everything works.
  7. Finally, set the DNS of any machine in the user file to your server IP and any listed hosts will be forwarded.
